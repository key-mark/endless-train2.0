## 1. 项目定位

《无限列车 SLG》是一款以末日列车为核心基地的轻 SLG 生存成长游戏。

Demo 的目标不是制作完整商业 SLG，而是验证最核心的前期闭环：

```text
列车主界面 → 前往废弃站点 → 竖屏车道防守战斗 → 获得资源 → 返回列车 → 升级列车 / 炮台 → 挑战下一站
```

玩家经营的不是固定城市，而是一列可以移动、战斗、扩建和成长的末日列车。

本版本的重点更新是：

- 废弃站点战斗不再采用“敌人从右侧向左靠近列车”的横向战斗。
- 战斗改为 `540 x 960` 竖屏双轨道防守。
- 玩家防守屏幕底部的横向列车基地。
- 敌人和升级目标从屏幕上方向下移动。
- 玩家通过左右拖动移动列车上的英雄和炮台。
- 升级目标必须被子弹击毁后才会获得奖励。
- 普通波次结束后进入 Boss 战。

## 2. 核心设计目标

### 2.1 Demo 目标

- 在最短时间内跑通可玩闭环。
- 让玩家清楚理解“列车是基地”。
- 让玩家清楚感受到“出战获得资源，返回后升级列车”。
- 用竖屏双轨道战斗验证前期操作体验。
- 用升级目标验证“清敌人与抢成长”的战斗取舍。
- 用 Boss 战作为单局明确收尾。

### 2.2 战斗目标

废弃站点战斗主要验证以下循环：

```text
移动炮台 → 射击敌人和升级目标 → 玩家局内变强 → 敌人数量随输出增长 → 击败 Boss
```

战斗不是纯挂机，也不是复杂 RTS。玩家只做一个核心操作：

```text
左右拖动火力位置
```

但这个操作会带来持续判断：

- 现在应该打左轨敌人，还是右轨敌人？
- 应该先清敌人保命，还是先打爆升级目标变强？
- Boss 预警出现后，要不要移动到安全轨道？
- 当前火力是否足够贪升级目标？

## 3. 整体玩法结构

### 3.1 外层：列车主界面

列车主界面承担 SLG 前期成长功能。

玩家在这里看到：

- 当前列车外观。
- 当前资源。
- 当前列车等级。
- 当前炮台等级。
- 当前站点进度。
- 出战按钮。
- 升级 / 修理按钮。

玩家在这里做长期决策：

```text
是否升级炮台？
是否修理列车？
是否升级车体？
是否前往下一个废弃站点？
```

### 3.2 内层：废弃站点战斗

废弃站点战斗承担操作体验和短局成长功能。

玩家进入战斗后：

1. 屏幕底部出现横向列车基地。
2. 英雄和炮台位于列车上方或列车平台处。
3. 两条轨道从屏幕上方向下延伸到列车前方。
4. 敌人和升级目标沿轨道下落。
5. 玩家左右拖动英雄和炮台，对准目标。
6. 英雄和炮台自动向上开火。
7. 击毁敌人获得少量资源或击杀计数。
8. 击毁升级目标获得本局临时强化。
9. 普通波次结束后，Boss 出现在屏幕上方。
10. 击败 Boss 后结算资源，返回列车主界面。

## 4. 画面分辨率与显示基准

### 4.1 内部玩法分辨率

本 Demo 的内部玩法分辨率固定为：

```text
540 x 960
```

这是所有玩法坐标、UI 排版、轨道位置、角色大小和战斗距离的唯一设计基准。

### 4.2 屏幕比例

```text
9:16 竖屏
```

战斗画面必须优先适配手机竖屏。

### 4.3 桌面测试窗口

桌面测试时可以使用放大窗口：

```text
810 x 1440
```

但这只是显示缩放，不是玩法分辨率。

### 4.4 关键规则

- 不要为了桌面窗口修改玩法坐标。
- 不要按 2K 屏幕重新排 UI。
- 所有战斗对象尺寸都以 `540 x 960` 为准。
- 美术资源可以更高清，但摆放和视觉占比必须服从 `540 x 960`。
- UI 和战斗区域必须按手机竖屏阅读习惯设计。

## 5. 战斗画面构图

### 5.1 纵向区域划分

以 `540 x 960` 为基准，战斗画面建议分为四个纵向区域：

| 区域 | y 范围 | 功能 |
|---|---:|---|
| Boss / 远景威胁区 | 0 - 160 | Boss、远处轨道、顶部环境压迫 |
| 主战斗区 | 160 - 700 | 敌人、升级目标、子弹飞行、轨道压力 |
| 英雄判定区 | 700 - 820 | 英雄站位、敌人第一伤害判定、玩家操作焦点 |
| 列车基地区 | 820 - 960 | 横向列车基地、炮台安装区、底部 UI 安全区 |

### 5.2 横向布局

推荐双轨道位置：

| 轨道 | x 坐标 |
|---|---:|
| 左轨 | 175 |
| 右轨 | 365 |

玩家移动范围建议：

| 范围 | x 坐标 |
|---|---:|
| 最左 | 52 |
| 最右 | 488 |

设计说明：

- 敌人、升级目标、Boss 预警主要沿两条轨道出现。
- 玩家不强制吸附在轨道上，可以在左右范围内自由拖动。
- 当前玩家更接近哪条轨道，就更容易清理该轨道威胁。
- 两条轨道必须在竖屏中一眼可见，不能被背景雪雾或 UI 淹没。

## 6. 玩家操作设计

### 6.1 核心操作

玩家只需要执行一个主要操作：

```text
左右拖动
```

拖动影响：

- 英雄横向位置。
- 炮台横向位置。
- 自动开火的火力中心。
- 当前主要防守轨道。

### 6.2 不做的操作

Demo 阶段不需要：

- 手动点击射击。
- 多单位框选。
- 复杂技能轮盘。
- 多炮台分别控制。
- 建筑摆放。
- 士兵编队。

### 6.3 操作体验目标

操作必须满足：

- 单指可玩。
- 拖动反应直接。
- 火力位置变化清楚。
- 玩家能明显感到“我正在把炮台推向左轨 / 右轨”。
- 操作失败主要来自判断错误，而不是输入复杂。

## 7. 战斗核心机制

### 7.1 敌人下落

敌人从屏幕上方出现，沿左右两条轨道向下移动。

敌人目标不是横向接近列车，而是纵向突破防线：

```text
屏幕上方生成 → 沿轨道向下移动 → 到达英雄判定线 → 继续突破到列车线
```

敌人造成伤害分两层：

| 位置 | 结果 |
|---|---|
| 到达英雄判定线并与英雄区域重合 | 对玩家造成完整伤害，敌人移除 |
| 穿过英雄后到达列车线 | 对列车造成较低伤害，敌人移除 |

设计意图：

- 英雄线是第一道防线。
- 列车线是最终底线。
- 玩家横向拖动可以主动拦截当前轨道威胁。

### 7.2 自动射击

英雄和炮台会自动向上开火。

射击规则：

- 子弹从玩家当前位置向上发射。
- 子弹优先命中同轨道或附近目标。
- 玩家不需要点击开火。
- 火力价值来自玩家的横向站位选择。

### 7.3 升级目标

升级目标是战斗中的可破坏奖励物。

重要规则：

```text
升级目标不是拾取物。
升级目标必须被子弹击毁后才会生效。
```

升级目标和敌人一样从屏幕上方出现，并沿轨道向下移动。

它的作用是制造战斗取舍：

```text
打敌人 = 保命
打升级目标 = 变强
```

如果升级目标血量高、移动快，玩家就必须投入更多火力；如果此时敌人压力也大，玩家就要在安全和收益之间做判断。

### 7.4 局内成长

升级目标提供本局临时强化。

示例强化：

| 类型 | 效果 |
|---|---|
| 攻击提升 | 本局攻击力增加 |
| 射速提升 | 本局射击频率提升 |
| 回复 | 恢复列车或玩家 HP |
| 资源 | 获得少量废料或金币型资源 |
| 攻击倍率 | 本局总伤害倍率提升 |
| 炮台倍率 | 本局炮台伤害提升 |
| 子弹增加 | 本局炮台多发射子弹 |
| 爆炸弹 | 本局子弹获得范围伤害 |

战斗结束后，这些临时强化清空。

### 7.5 动态压力

当玩家通过升级目标变强后，敌人数量也可以逐渐增加。

设计目标不是惩罚玩家，而是让成长更有爽感：

- 玩家变强后，能看到更多敌人被快速清掉。
- 敌人数量增加，让火力成长可见。
- 如果场上敌人已经过多，压力应放缓，避免失控堆怪。

## 8. Boss 战设计

### 8.1 Boss 定位

Boss 是废弃站点战斗的最终收尾。

Boss 不应该像普通敌人一样下落撞列车，而是位于屏幕上方，作为大型压迫源存在。

推荐 Boss 主题：

```text
废弃站点装甲列车
```

或后续扩展为：

- 掠夺者重型列车。
- 冰封防御机械。
- 感染工程车。
- 失控轨道炮平台。

### 8.2 Boss 行为

Boss 核心行为：

- 多条血量。
- 固定在屏幕上方。
- 向左右轨道释放预警攻击。
- 阶段变化后攻击频率提升。
- 可召唤少量普通敌人。

### 8.3 轨道预警

Boss 攻击前必须显示明显预警。

预警设计要求：

- 预警覆盖对应轨道。
- 颜色必须明显，例如半透明红色或橙红色。
- 不能完全遮挡敌人和子弹。
- 玩家需要有短暂反应时间移动到安全区域。

### 8.4 Boss 阶段

Demo 可做三段血条：

| 阶段 | 体验目标 | 行为 |
|---|---|---|
| 阶段 1 | 让玩家理解 Boss 预警 | 随机单轨攻击 |
| 阶段 2 | 提升压迫 | 攻击间隔缩短，召唤基础敌人 |
| 阶段 3 | 最终爆发 | 左右交替攻击，召唤快速敌人 |

## 9. 列车主界面与战斗的融合

### 9.1 外层长期成长

列车主界面中的升级是永久成长。

永久成长包括：

- 炮台等级。
- 列车等级。
- 列车最大 HP。
- 列车外观等级。
- 后续可扩展车厢数量。

### 9.2 内层临时成长

废弃站点战斗中的升级目标是本局成长。

本局成长包括：

- 当前战斗攻击力提升。
- 当前战斗射速提升。
- 当前战斗多子弹。
- 当前战斗爆炸弹。
- 当前战斗恢复。

### 9.3 两种成长的关系

| 成长 | 获得方式 | 是否保留 | 玩家感受 |
|---|---|---|---|
| 永久成长 | 返回列车后消耗资源升级 | 保留 | 我的列车越来越强 |
| 局内成长 | 战斗中射爆升级目标 | 本局有效 | 这一局越打越爽 |

设计重点：

- 不要把战斗升级目标做成永久升级。
- 不要把列车主界面升级做成战斗内随机奖励。
- 两者要共同服务“出战—回收—升级—再出战”的循环。

## 10. 资源与奖励设计

### 10.1 推荐资源

Demo 外层建议保留四种资源：

| 资源 | 用途 |
|---|---|
| 废料 | 升级炮台、修理列车、升级车体 |
| 燃料 | 前往废弃站点 |
| 食物 | 后续幸存者系统预留，Demo 可只展示 |
| 零件 | Boss 掉落，用于高级升级 |

### 10.2 战斗内奖励

战斗内可以显示：

- 击杀数。
- 本局获得废料。
- Boss 掉落零件。
- 用时。
- 通关站点编号。

如果使用“金币”作为战斗内显示资源，建议结算时转换成废料，避免 Demo 资源体系过散。

### 10.3 胜利结算

击败 Boss 后显示胜利面板。

胜利面板展示：

- 击杀数。
- 获得废料。
- 获得燃料。
- 获得食物。
- 获得零件。
- 用时。
- 返回列车按钮。

### 10.4 失败结算

列车 HP 归零或玩家 HP 归零后显示失败面板。

失败面板展示：

- 击杀数。
- 存活时间。
- 是否保留少量战利品。
- 返回列车按钮。

Demo 简化建议：失败不推进站点，不发 Boss 奖励。

## 11. 视觉设计规范

### 11.1 列车基地

列车位于画面底部，是玩家必须保护的核心。

视觉要求：

- 横向展开，占据底部主要宽度。
- 不能像普通背景装饰，必须明显是玩家基地。
- 炮台要看起来安装在列车上，而不是悬浮。
- 列车升级后应有明显外观变化。

### 11.2 炮台

炮台是玩家操作反馈核心。

视觉要求：

- 跟随玩家拖动左右移动。
- 炮管朝上。
- 底座与列车连接明确。
- 射击时有枪口火光或后坐力反馈。
- 升级后可以变大、变亮、增加炮管或增加子弹特效。

### 11.3 英雄

英雄是玩家位置的人形锚点。

视觉要求：

- 位于英雄判定线附近。
- 和炮台共享横向位置。
- 看起来站在列车平台或炮台附近。
- 后续可以扩展为“带领士兵”的视觉中心。
- 升级或局内成长后，可以增加随从士兵数量，形成夸张视觉反馈。

### 11.4 敌人

敌人必须和升级目标一眼区分。

视觉要求：

- 威胁感强。
- 轮廓清晰。
- 沿轨道向下压迫。
- 类型之间有明显差异：基础、快速、重甲。
- 受击时有闪烁、抖动或血条反馈。

### 11.5 升级目标

升级目标必须看起来是“可以射击的奖励物”，不是被动拾取物。

视觉要求：

- 有奖励感，但也有可破坏感。
- 可以显示小血条或破损状态。
- 和敌人颜色、轮廓、动效明显不同。
- 需要让玩家一眼知道“打爆它会变强”。

推荐表现：

- 空投补给箱。
- 发光能源核心。
- 漂浮改装模块。
- 军火箱。
- 带图标的强化装置。

### 11.6 Boss

Boss 必须是画面上方最大视觉焦点。

视觉要求：

- 体积明显大于普通敌人。
- 有多条血量或阶段变化。
- 阶段变化时外观或颜色变化。
- 攻击前有轨道预警。
- 被击败时有明显爆炸或崩坏反馈。

### 11.7 轨道与预警

轨道是战斗阅读的基础。

视觉要求：

- 左右两条轨道清晰可见。
- 轨道不能太细，否则玩家难以判断敌人所在列。
- 预警必须明显，但透明度不能过高遮挡战斗对象。
- 轨道可以有雪地、铁轨、废弃站台、警示灯等末日氛围元素。

## 12. UI 设计规范

### 12.1 战斗 UI

战斗 UI 必须轻量，不能遮挡轨道判断。

推荐显示：

- 当前 HP。
- 当前波次 / Boss 状态。
- 击杀数。
- 当前局内强化摘要。
- Boss 血条。

UI 布局建议：

| UI | 推荐位置 |
|---|---|
| HP | 左上角 |
| 当前站点 / 波次 | 上方中间 |
| 资源 / 击杀数 | 右上角 |
| 局内强化摘要 | 左侧或右侧小图标 |
| Boss 血条 | Boss 出现时置于顶部或 Boss 下方 |
| 胜利 / 失败面板 | 屏幕中心 |

### 12.2 主界面 UI

列车主界面必须清楚展示长期成长。

推荐显示：

- 资源栏。
- 列车预览。
- 列车等级。
- 炮台等级。
- 当前站点。
- 升级按钮。
- 修理按钮。
- 前往废弃站点按钮。

## 15. 验收标准

Demo 完成后应满足：

1. 启动后进入列车主界面。
2. 玩家能看到资源、列车等级、炮台等级和当前站点。
3. 玩家能点击“前往废弃站点”进入竖屏战斗。
4. 战斗内部玩法分辨率以 `540 x 960` 为基准。
5. 屏幕底部能看到横向列车基地。
6. 玩家能左右拖动英雄和炮台。
7. 英雄和炮台会自动向上射击。
8. 敌人会从屏幕上方向下沿两条轨道移动。
9. 升级目标会从屏幕上方向下移动。
10. 升级目标必须被子弹击毁后才会生效。
11. 玩家能在“清敌人”和“打升级目标”之间做选择。
12. 普通波次结束后出现 Boss。
13. Boss 位于屏幕上方，不作为普通下落敌人。
14. Boss 会释放轨道预警攻击。
15. 击败 Boss 后出现胜利结算。
16. 失败后出现失败结算。
17. 返回列车后可用资源升级炮台或列车。
18. 第二次进入战斗时，永久升级带来的火力提升能被感知。
