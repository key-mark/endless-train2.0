# AGENTS.md

## 项目说明

本项目是使用 Godot 4 和 C# 开发的《无限列车 SLG》1 天可玩 Demo 原型。

Demo 的目标不是制作完整商业 SLG，而是验证一个最小但完整的前期玩法闭环：

```text
列车主界面 → 前往废弃站点 → 竖屏双轨道防守战斗 → 击败 Boss → 获得资源 → 返回列车 → 升级炮台 / 列车 → 下一局更强
```

玩家经营的不是固定城市，而是一列可以移动、战斗、扩建和成长的末日列车。

---

## 技术要求

- 使用 Godot 4.x。
- 使用 C# 编写玩法脚本。
- 不要把项目改成 GDScript。
- 不要引入不必要的复杂架构。
- 不要提前制作完整商业 SLG 系统。
- 当前阶段优先保证 Demo 可运行、可验收、可快速迭代。

## Demo 核心循环

必须围绕以下循环开发：

```text
1. 玩家进入列车主界面。
2. 查看废料、燃料、食物、零件等资源。
3. 查看列车等级、炮台等级和当前站点。
4. 玩家可以升级炮台或修理列车。
5. 玩家点击“前往废弃站点”进入战斗。
6. 战斗中玩家左右拖动英雄和炮台。
7. 英雄和炮台自动向上射击。
8. 敌人和升级目标从屏幕上方向下移动。
9. 玩家需要在清理敌人和射击升级目标之间分配火力。
10. 普通波次结束后 Boss 出现。
11. 击败 Boss 后获得资源奖励。
12. 返回列车主界面。
13. 使用资源升级炮台或列车。
14. 下一次战斗明显变强。
```

---

## 战斗玩法规则

废弃站点战斗必须是竖屏双轨道防守玩法。

### 基础要求

- 内部玩法分辨率为 `540 x 960`。
- 屏幕比例为 `9:16`。
- 战斗画面为竖屏。
- 底部是固定不动的横向列车基地。
- 屏幕中有两条清晰可见的纵向轨道。
- 敌人从屏幕上方生成，并沿轨道向下移动。
- 升级目标从屏幕上方生成，并沿轨道向下移动。
- 玩家通过左右拖动控制英雄和炮台的横向位置。
- 英雄和炮台共享同一个横向位置。
- 英雄和炮台自动向上开火。
- 玩家不需要点击开火。
- 玩家不需要控制列车移动。

### 禁止误解

不要把战斗做成：

```text
敌人从右侧出现 → 向左靠近列车
```

正确方向是：

```text
敌人 / 升级目标从上方出现 → 向下压向底部列车
```

## 画面分辨率规则

所有玩法坐标、UI 排版、轨道位置、敌人移动、列车位置、Boss 位置，都应以以下内部玩法分辨率为准：

```text
540 x 960
```

桌面测试窗口可以放大显示，例如：

```text
810 x 1440
```

但不要为了桌面显示修改玩法坐标。

推荐画面区域划分：

```text
0 - 160       Boss / 远景威胁区
160 - 700     敌人、升级目标、子弹飞行区
700 - 820     英雄判定线 / 玩家操作核心区
820 - 960     横向列车基地 / 炮台安装区 / 底部 UI 安全区
```

推荐轨道位置：

```text
左轨 x = 175
右轨 x = 365
```

玩家可以在一定横向范围内自由拖动，不需要强制吸附到轨道中心。

## 升级目标规则

升级目标不是普通拾取物。

升级目标必须满足：

- 从屏幕上方向下移动。
- 和敌人有明显不同的视觉表现。
- 拥有 HP。
- 必须被子弹击毁后才会生效。
- 玩家碰到升级目标不会直接获得升级。
- 如果升级目标离开屏幕或未被击毁，则不会获得奖励。

升级目标的设计目的：

```text
让玩家在“清敌人保命”和“打升级变强”之间做选择。
```

至少需要支持以下局内升级：

| 类型 | 效果 |
|---|---|
| attack_add | 本局攻击力增加 |
| fire_rate | 本局射速提升 |
| heal | 恢复列车 HP |
| bullet_add | 本局炮台子弹数量增加 |
| explosive | 本局炮弹获得范围伤害，可选 |

战斗结束后，所有局内临时升级都必须清空。

---

## 敌人规则

敌人必须：

- 从屏幕上方生成。
- 沿左轨或右轨向下移动。
- 被子弹命中后扣血。
- HP 归零后死亡并移除。
- 到达英雄判定线时，如果和英雄横向位置足够接近，则对玩家造成完整伤害。
- 穿过英雄判定线后继续下落。
- 到达列车受击线时，对列车造成较低伤害并移除。

Demo 阶段至少需要一种基础敌人。

如果时间允许，可以扩展为：

| 类型 | 定位 |
|---|---|
| Grunt | 基础敌人 |
| Runner | 快速低血敌人 |
| Shield | 慢速高血敌人 |

---

## Boss 规则

Boss 是废弃站点战斗的收尾挑战。

Boss 必须：

- 在普通波次之后出现。
- 位于屏幕上方。
- 不沿轨道下落。
- 拥有多条血量，建议至少 3 条。
- 使用左轨或右轨的轨道预警攻击。
- 预警出现后短暂延迟，再造成伤害。
- 玩家位于危险轨道附近时受到伤害。
- 被击败后触发胜利结算。

Boss 可以召唤敌人，但 Demo 阶段可以先做简化版本。

## 列车主界面规则

列车主界面是游戏外层成长界面。

必须显示：

- 废料 Scrap
- 燃料 Fuel
- 食物 Food
- 零件 Parts
- 列车等级 TrainLevel
- 炮台等级 CannonLevel
- 当前站点 CurrentStation
- 列车 HP
- 升级炮台按钮
- 修理列车按钮，可选
- 前往废弃站点按钮

列车主界面必须能进入战斗。

战斗胜利返回后，资源和站点进度必须更新。

---

## 长期成长规则

Demo 阶段只需要最小长期成长。

必须实现：

```text
炮台升级
```

炮台升级应影响下一次战斗，例如：

- 提高基础伤害。
- 缩短射击间隔。
- 或增加火力表现。

可选实现：

```text
列车升级
修理列车
列车 Lv1 / Lv2 / Lv3 外观切换
```

不要实现复杂科技树、建筑队列、装备系统或英雄养成。

## 资源规则

Demo 使用以下资源：

| 资源 | 用途 |
|---|---|
| 废料 Scrap | 升级炮台、修理列车、升级车体 |
| 燃料 Fuel | 前往废弃站点 |
| 食物 Food | 后续幸存者系统预留，Demo 可只显示 |

当前阶段不需要金币系统。

如果旧代码或旧文档中存在金币，可以暂时映射为废料，避免资源系统过多。

## 胜负规则

### 胜利

当 Boss 被击败时：

- 显示胜利结算面板。
- 显示获得的废料、燃料、食物、零件。
- 玩家点击“返回列车”。
- 发放奖励。
- 当前站点 `CurrentStation + 1`。
- 返回列车主界面。

### 失败

当列车 HP 小于等于 0 时：

- 显示失败面板。
- 停止战斗。
- 不发放通关奖励。
- 不推进站点。
- 玩家点击“返回列车”后回到列车主界面。

## 美术和占位规则

当前阶段使用灰盒和占位图形即可。

优先使用：

- ColorRect
- Label
- 简单 Sprite2D
- 简单几何图形
- 清晰文字提示

所有核心视觉对象应保留可替换结构：

```text
VisualRoot
```

后续正式美术应只替换 `VisualRoot` 下的内容，不应影响玩法逻辑。

### 视觉要求

- 炮台必须看起来安装在列车上，而不是悬浮在列车上方。
- 英雄应位于英雄判定线附近。
- 英雄和炮台共享横向位置。
- 敌人和升级目标必须一眼能区分。
- 升级目标应看起来像“可以射击的奖励物”。
- Boss 应该比普通敌人大，并且有明显血条。
- 轨道预警必须明显，但不能完全遮挡子弹、敌人和升级目标。

按照以下顺序开发，不要跳步：
